# 마이링크 (MyLink) - 사용자 시나리오

## 1. 개요
* **목표**: 마이링크 서비스의 주요 사용자 흐름(Flow)을 정의합니다.
* **사용자 역할**:
  * **크리에이터 (Creator)**: 마이링크에 가입하여 자신의 링크 페이지를 구성하고 관리하는 회원
  * **방문자 (Visitor)**: 크리에이터가 공유한 링크를 통해 프로필 페이지에 접속하는 일반 사용자

## 2. 주요 시나리오

### SC-01: 회원가입 및 초기 온보딩 (크리에이터)
1. 크리에이터가 마이링크 랜딩 페이지에 접속하여 '시작하기' 버튼을 클릭합니다.
2. 인증 페이지에서 **구글 로그인(Google OAuth)**을 클릭하여 가입을 진행합니다 (MVP 단계).
3. 최초 로그인 시, 자신만의 고유한 프로필 주소를 만들기 위해 **아이디(Username)**를 입력합니다.
   * *시스템: `admin`, `login`, `api`, `settings` 등 사전 정의된 예약어인지 검사하고, 영문/숫자 조합 및 길이를 검증하여 사용 가능 여부를 실시간으로 알려줍니다.*
4. 사용 가능한 아이디가 확인되면 회원가입이 완료되고 빈 대시보드 화면으로 이동합니다.

### SC-02: 링크 추가 및 관리 (크리에이터)
1. 대시보드 좌측의 '링크 탭'에서 '새로운 링크 추가하기' 버튼을 누릅니다.
2. 폼이 나타나면 링크의 제목(예: "내 깃허브")과 URL(예: "https://github.com/poatan2")을 입력합니다.
3. (선택 사항) 좌측 썸네일 아이콘을 업로드하거나 기본 제공 아이콘(GitHub 로고 등)을 선택합니다.
4. 추가된 여러 링크 항목들은 마우스로 드래그 앤 드롭하여 위아래 노출 순서를 변경할 수 있습니다.
5. 링크 우측의 토글(스위치)을 통해 언제든지 해당 링크를 임시로 숨기거나 다시 노출할 수 있습니다.
6. **(PC 화면 뷰어)**: 대시보드 우측 영역에는 수정 중인 프로필 페이지의 **실시간 뷰어(Preview)**가 띄워져 있어, 저장 버튼을 누르지 않아도 변경된 사항을 즉시 확인할 수 있습니다.

### SC-03: 디자인 및 프로필 커스터마이징 (크리에이터)
1. 대시보드 상단의 '디자인 탭'으로 이동합니다.
2. 프로필 상단에 노출될 프로필 이미지 업로드 버튼을 눌러 사진을 등록하고, 화면에 보일 닉네임과 짧은 소개글을 작성합니다.
3. 배경색(또는 배경 이미지)과 링크 버튼의 스타일(Shadcn/ui 기반의 모던/미니멀 스타일)을 선택합니다.
4. 우측 뷰어에서 자신의 디자인이 어떻게 반영되는지 실시간으로 확인합니다.

### SC-04: 애널리틱스 통계 확인 (크리에이터) - Level 2
1. 대시보드 상단의 '통계(Analytics) 탭'으로 이동합니다.
2. 화면 상단에 큼지막한 카드로 '내 프로필 총 방문자 수'와 '총 링크 클릭 수'가 직관적으로 표시됩니다.
3. 그 아래에는 최근 7일(또는 30일) 간의 일별 방문자 추이를 보여주는 **꺾은선 그래프(Chart)**가 렌더링 됩니다.
4. 차트 아래쪽에는 링크별 클릭 수가 리스트 형태로 나열되어, 어떤 링크의 전환율이 가장 높은지 구체적으로 파악합니다.

### SC-05: 링크 페이지 방문 (방문자) - PC 기준
1. 방문자가 크리에이터가 공유한 링크(`예: mylink.com/poatan2`)를 클릭하여 데스크탑 브라우저로 접속합니다.
2. 화면 중앙에 크리에이터의 프로필과 소개글이 위치하며, 그 아래 등록된 링크들이 깔끔한 버튼 형태로 나열되어 있습니다 (전체적으로 PC 화면의 넓은 공간을 활용하여 중앙 정렬됨).
3. 특정 링크 버튼을 클릭하면 해당 외부 웹사이트로 새 탭이 열립니다.
4. *시스템: 백그라운드에서 해당 링크의 클릭 수 카운트와 프로필 뷰 카운트를 1씩 증가시켜 Firebase Firestore에 기록합니다.*
